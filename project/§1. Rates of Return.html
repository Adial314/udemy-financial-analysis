Simple Rate of Return (SROR)¶

The difference between the Close and the Adj Close is that the adjusted closing value takes into account dividends paid to investors as well as stock splits. We will use this value for our calculations.

The simple rate of return is best applied when studying several securities over the same time-span.

The simple rate of return is the difference between our present value and our initial investment divided by our initial investment like so:

$ \text{SROR} = \frac{P_{\text{Val}} - P_{\text{Inv}}}{P_{\text{Inv}}} = \frac{P_{\text{Val}}}{P_{\text{Inv}}} - 1 $

Let's run this calculation

Logarithmic Rate of Return (LROR)¶

The logarithmic rate of return is best used when studying a single security and comparing its return over several time-periods, as opposed to comparing several securities against each other.

$ \text{LROR} = ln\left( \frac{P_{t}}{P_{t-1}} \right) $

Portfolios Rates of Return¶

Portfolios are often composed of multiple assets and asset classes. The rates fo return for each asset is determined and the portfolio's rate of return becomes the vector sum of the weight of the asset and the asset's ROR.

$ \text{Portfolio ROR} = \sum^{Assets} \left( \text{Asset ROR} \cdot \text{Asset Weight} \right) $

For example, consider the following portfolio:

Security Weight Ave. Annual ROR
APPL 40 % 11 %
PG 30 % 11 %
MSFT 5 % 9 %
GE 25 % 7 %

The the portfolio's over-all ROR would be:

$ \text{Portfolio ROR} = (\omega_{\text{APPL}} \cdot \text{ROR}_{\text{APPL}}) + (\omega_{\text{PG}} \cdot \text{ROR}_{\text{PG}}) + (\omega_{\text{MSFT}} \cdot \text{ROR}_{\text{MSFT}}) + (\omega_{\text{GE}} \cdot \text{ROR}_{\text{GE}}) = (0.40 \cdot 11) + (0.30 \cdot 11) + (0.05 \cdot 9) + (0.25 + 7) = 9.9 \% $

Let's run this calculation using python:

Before we visually study the returns of the individual securities within the portfolio, we have to normalize the adjusted close pricing with respect to its price at the beginning of our timeframe. This will allow us to equitably compare the securities within the portfolio. We will divide each price by the start of the dataset and multiply is by 100 like so:

$ \frac{P_i}{P_0} \cdot 100 $

We can plot the resulting data with confidence that it is representative.

In [9]:
# Call plotting on hypothetical normalized dataframe
(Port / Port.iloc[0] * 100).plot(figsize = (15, 6));

# Illustrate
plt.show()

We can see that the data points all begin at 100, indicating that they have been normalized because $(P_0 / P_0) \cdot 100 = 1 \cdot 100 = 100$. We have successfully normalized the data.

Let's create a dataframe for the simple rates of return of each stock and define the weights for the securities within the portfolio. Then the portfolio return will be simply the sum of products of the security returns and weights of the securities.

Remember, we are using the simple rate of return because we are studying multiple securities under the same time frame. The logarithmic rate of return is best used for studying a single security's performance.

# Calculate the annualized SROR
annual_SROR = round((SROR.mean() * 250), 5) * 100

# Define security weights within the portfolio
weights = np.array([0.25, 0.25, 0.25, 0.25])

# Check if the weights are balanced
if weights.sum() != 1: print('Error: Weights do not add up.')
else: pass

# ---------- Calculate the portfolio SROR ---------- #

# Instantiate at 0
Port_SROR = 0

# Cycle through weights and take a vector sum
for i in range(len(weights)):
    temp = (weights[i] * annual_SROR.iloc[i])
    Port_SROR = Port_SROR + temp

# Print resulting Portfolio SROR
print('Portfolio Simple ROR: ', Port_SROR, '%')

Portfolio Simple ROR:  13.76725 %

When determining if your invesments are extraordinary, you must compare the rate of return of your invesments against an the market as a whole.
